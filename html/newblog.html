New features in CSS3

There are some significant changes in the transition to CSS3. The following points the specific technical changes introduced.

  1. Multiple background
    2. A background property that is introduced in css3 is the ability to add multiple background images on one line using the background attribute. Inorder to make changes to the background images, adding background style attributes like position, fixed, repeat can come after each image before the comma.
  2. Selectors
    3. Besides the old selectors in CSS2 three more attribute selectors are introduced.
    1. Element [attribute^="value"] Selector

      This selector applies the styles below it on elements with an attribute whose attributes start with value specified.

    2. Element [attribute$="value"] Selector

      This selector applies the styles below it on elements with an attribute whose attributes ends with value specified.

    3. Element [attribute*="value"] Selector

      This selector applies the styles below it on elements with an attribute whose attributes includes value specified.

  3. Pseudo-classes
    4. The following pseudo-classes were added to css3.
    1. :root

      This selector allows to select the root element in our DOM hierarchy ("html"for webpages).since CSS can be used for xml and svg too, it doesn’t necessarily mean html".

    2. :last-child

      Using :last-child the last child of a given selector can be modified.

  4. Colors
    5. Besides the old hexadecimal and rgb ways to add colors, CSS3 brings the following two ways that gives us more control over colors.
    1. RGBA

      It is the same as rgb but accepts a third parameter to control the opacity of the color we have chosen. The fourth argument is between 1 and 0(where 0 is transparent and 1 is opaue).

    2. Opacity

      If we don’t want to use the rgba, the opacity option lets us control the opacity of the color after specifying it with rgb or hexadecimal notation.

    3. HSL and HSLA

      These are important In order to change the hue, saturation and lightness. The hsl function accepts three parameters as degree for the hue,percentage for saturation and lightness.

  5. Drop shadows
    6. These styles add a dropping shadow to an image. The inputs are specifies as the box shadow property that control the color, width, height, blue and offset of the shadow.
  6. Text shadows
    7. Similar to the drop shadows for texts. These attribute accepts X and Y offsets from the element, blur radius, and color.
  7. border radius
    8. Border radius lets us control the radius(roundedness) of corners. It can accepts one input that lets it to be used for every corner or accepts four inputs if we want to use different corner radiuses.

Why relative size matters in CSS?

Back then, when there were no differences between size of computers, using pixel-based styling in our css never raised an issue. But when computers started to come in different sizes, actual measured(pixel) stylings started to show different styles on different machines. These is one of the reasons why relative sizes should be used. The other important thing to consider is browser zooming.

Browzer zooming

When browser zooming is used, it scales up the whole content of the page in proportion. That means it won’t have much difference whether we used relative or fixed measurements. But here let’s consider the type of browser zooming that enables us to adjust preferred font size. Accommodating this feature is important if short sighted people are to view the site.
